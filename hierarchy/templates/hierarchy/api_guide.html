API guide

JSON endpoints served by this site. Base URL: {{ site_api_base }}

POST /api/auth/users/

Create a user and employee profile. Body: JSON. CSRF exempt.

  • Required fields: samAccountName, password, givenName, surname, tenant (tenant slug).
  • Responses: 201 created; 400 validation / bad tenant; 409 username exists.
curl -sS -X POST "{{ site_api_base }}/api/auth/users/" \
  -H "Content-Type: application/json" \
  -d '{"samAccountName":"jdoe","password":"SecurePass1!","givenName":"Jane","surname":"Doe","tenant":"default"}'

POST /api/auth/login/

Login with JSON; response sets a session cookie (sessionid). CSRF exempt.

  • Body: username, password.
  • Responses: 200 ok; 401 bad credentials; 403 disabled.
curl -sS -c cookies.txt -X POST "{{ site_api_base }}/api/auth/login/" \
  -H "Content-Type: application/json" \
  -d '{"username":"jdoe","password":"SecurePass1!"}'

GET /api/employees/

One employee plus signature images (Base64). Query string only.

  • Query: tenant_id (required) and exactly one of user_id, username, civil_id.
  • Auth (any one): Authorization: Bearer token ({% if employee_api_token_configured %}set on server{% else %}optional / may be unset{% endif %}); or tenant API key header (default X-Api-Key, configurable per tenant); or staff session cookie.
  • Typical errors: 400 bad query; 401 / 403; 404 not found; 409 duplicate civil_id.
curl -sS "{{ site_api_base }}/api/employees/?tenant_id=1&username=jdoe" \
  -H "X-Api-Key: <secret>"

Auth notes (GET /api/employees/)

  • Bearer secret: environment EMPLOYEE_API_TOKEN.
  • Tenant key: tenant settings or TENANT_API_KEY_<tenant_id>; header name: tenant settings or TENANT_API_KEY_HEADER_<tenant_id> (default header X-Api-Key).
