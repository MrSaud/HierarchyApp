JSON endpoints implemented by this Hierarchy app (routes in hierarchy/urls.py under /api/). Base URL: {{ site_api_base }}

Integrations should call versioned /api/v1/… reads for stable JSON, weak ETag / If-None-Match (304), and bulk PATCH with If-Match (412 when stale). {% trans "Authentication" %}

Protected endpoints accept any one of the following. The target tenant_id (or tenant slug on create-user) must match the credential’s tenant unless you are a superuser staff session.

  • Tenant API key — header X-Api-Key by default (configurable per tenant on {% trans "API access" %}). Required for unattended integrations.
  • Global Bearer — Authorization: Bearer <token> using EMPLOYEE_API_TOKEN ({% if employee_api_token_configured %}configured on server{% else %}not set — optional env override{% endif %}).
  • Staff session — cookie from POST /api/auth/login/ or the web UI; staff must belong to the same tenant (superusers may access any active tenant).

POST /api/auth/login/ itself does not require an API key; it returns a session for browser or staff API use.

{% trans "Response conventions" %}

  • Employee identity — Nested person objects (delegations, reports-to, assignment snapshots) include user_id, username, civil_id, and employee_number (strings; empty when not set). Full employee GET also returns these at the top level.
  • Auth user payloads — POST /api/auth/login/ includes civil_id and employee_number when a linked employee exists. POST /api/auth/users/ returns both fields on create (often empty until edited).
  • Versioned GET (/api/v1/…) — Successful reads include weak ETag, X-API-Version: 1, X-API-Read-Contract (stable payload name), and Link to this guide. Send If-None-Match to receive 304 when unchanged.
  • Bulk PATCH — Requires If-Match from the matching v1 GET snapshot; missing header → 428, stale tag → 412. Successful writes return a new ETag. Unit bulk locks against GET …/units/?format=flat only.
  • Assignments snapshot — Each row includes employee_id, user_id, username, civil_id, employee_number, position and date fields. Changing this shape changes ETags — refetch before bulk PATCH after upgrades.
